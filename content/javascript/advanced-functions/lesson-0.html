Recursion and Stack

Let’s return to functions and study them more in-depth. Our first topic will be recursion.

Recursion is a programming pattern useful when a task can be split into several smaller tasks of the same type.

What is Recursion?

When a function solves a task, it may call other functions during execution.

A special case occurs when a function calls itself. That is called recursion.

Recursion is useful when a task can be simplified into:

  • A simple action.
  • A smaller version of the same task.

Example: Power Function

Let’s write a function pow(x, n) that raises a number to a power. 


pow(2,2) = 4
pow(2,3) = 8
pow(2,4) = 16

Iterative Solution


function pow(x, n) {
  let result = 1;

  for (let i = 0; i < n; i++) {
    result *= x;
  }

  return result;
}

alert(pow(2,3));

Recursive Solution


function pow(x, n) {
  if (n == 1) {
    return x;
  } else {
    return x * pow(x, n - 1);
  }
}

alert(pow(2,3));

Recursive functions have two important parts:

  • Base case – the simplest case that stops recursion.
  • Recursive step – where the function calls itself with simpler input.

Example Execution


pow(2,4) = 2 * pow(2,3)
pow(2,3) = 2 * pow(2,2)
pow(2,2) = 2 * pow(2,1)
pow(2,1) = 2

The recursion reduces the problem step by step until reaching the base case.

Short Recursive Syntax


function pow(x, n) {
  return (n == 1) ? x : x * pow(x, n - 1);
}

Recursion Depth

The maximum number of nested calls is called recursion depth.

JavaScript engines limit recursion depth to prevent stack overflow.

Typical recursion depth limit is around 10000 calls depending on the engine.

Execution Context and Stack

Each function call has an execution context which stores:

  • Current variables
  • Current execution line
  • The value of this

Execution contexts are stored in a structure called the call stack.

Recursive Traversal

Recursion is useful for processing recursive data structures such as company departments. 


let company = {
  sales: [
    {name: "John", salary: 1000},
    {name: "Alice", salary: 1600}
  ],

  development: {
    sites: [
      {name: "Peter", salary: 2000},
      {name: "Alex", salary: 1800}
    ],

    internals: [
      {name: "Jack", salary: 1300}
    ]
  }
};

Recursive Salary Calculation


function sumSalaries(department) {

  if (Array.isArray(department)) {
    return department.reduce(
      (prev, current) => prev + current.salary,
      0
    );

  } else {
    let sum = 0;

    for (let subdep of Object.values(department)) {
      sum += sumSalaries(subdep);
    }

    return sum;
  }

}

alert(sumSalaries(company));

Recursive Data Structures

A recursive data structure is a structure that references itself.

Examples include:

  • HTML DOM tree
  • Company department structures
  • Linked lists

Linked List


let list = {
  value: 1,
  next: {
    value: 2,
    next: {
      value: 3,
      next: {
        value: 4,
        next: null
      }
    }
  }
};

Each element contains:

  • value
  • next reference to another list element

Linked List Creation


let list = { value: 1 };

list.next = { value: 2 };
list.next.next = { value: 3 };
list.next.next.next = { value: 4 };
list.next.next.next.next = null;

Prepending a Value


list = { value: "new item", next: list };

Removing an Element


list.next = list.next.next;

Summary

  • Recursion means a function calling itself.
  • Recursive functions must have a base case.
  • The recursion step breaks the problem into smaller parts.
  • Each recursive call creates an execution context stored in the stack.
  • Recursive data structures reference themselves.
  • Examples include trees, company structures and linked lists.
  • Recursive functions can usually be rewritten using loops.
  • Recursion often produces simpler and cleaner code.
