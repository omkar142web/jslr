Rest Parameters and Spread Syntax

Many JavaScript built-in functions support an arbitrary number of arguments.

Functions with Many Arguments

Some built-in functions accept any number of arguments.

  • Math.max(arg1, arg2, ..., argN) – returns the largest value.
  • Object.assign(dest, src1, ..., srcN) – copies properties.

In this chapter we will learn how to create functions like these and how to pass arrays to them.

Rest Parameters ...

A function can be called with any number of arguments regardless of how many parameters it defines. 


function sum(a, b) {
  return a + b;
}

alert(sum(1, 2, 3, 4, 5));

Extra arguments are ignored unless we capture them using rest parameters.

Rest parameters use three dots ... followed by a variable name.

Collecting Arguments into an Array


function sumAll(...args) {
  let sum = 0;

  for (let arg of args) {
    sum += arg;
  }

  return sum;
}

alert(sumAll(1));
alert(sumAll(1, 2));
alert(sumAll(1, 2, 3));

Mixing Regular and Rest Parameters


function showName(firstName, lastName, ...titles) {

  alert(firstName + " " + lastName);

  alert(titles[0]);
  alert(titles[1]);
  alert(titles.length);

}

showName("Julius", "Caesar", "Consul", "Imperator");

Rest Parameters Must Be Last


function f(arg1, ...rest, arg2) {
  // Error
}
Rest parameters must always appear at the end of the parameter list.

The arguments Object

JavaScript also provides a special array-like object called arguments.

It contains all arguments passed to the function. 


function showName() {

  alert(arguments.length);
  alert(arguments[0]);
  alert(arguments[1]);

}

showName("Julius", "Caesar");
showName("Ilya");

However, arguments is not a real array and does not support array methods like map().

Rest parameters are usually preferred in modern JavaScript.

Arrow Functions and arguments

Arrow functions do not have their own arguments object.

Instead, they use the arguments from the outer function. 


function f() {
  let showArg = () => alert(arguments[0]);
  showArg();
}

f(1);

Spread Syntax

Spread syntax also uses three dots ..., but it performs the opposite operation.

It expands an iterable into a list of arguments.

Example with Math.max


let arr = [3, 5, 1];

alert(Math.max(...arr));

Using Multiple Arrays


let arr1 = [1, -2, 3, 4];
let arr2 = [8, 3, -8, 1];

alert(Math.max(...arr1, ...arr2));

Combining Values


let arr1 = [1, -2, 3, 4];
let arr2 = [8, 3, -8, 1];

alert(Math.max(1, ...arr1, 2, ...arr2, 25));

Merging Arrays


let arr = [3, 5, 1];
let arr2 = [8, 9, 15];

let merged = [0, ...arr, 2, ...arr2];

alert(merged);

Spread with Strings


let str = "Hello";

alert([...str]);

Array.from Alternative


let str = "Hello";

alert(Array.from(str));

Array.from() works with both array-like objects and iterables, while spread works only with iterables.

Copying Arrays


let arr = [1, 2, 3];

let arrCopy = [...arr];

alert(arr === arrCopy);

arr.push(4);

alert(arr);
alert(arrCopy);

Copying Objects


let obj = { a: 1, b: 2, c: 3 };

let objCopy = { ...obj };

alert(obj === objCopy);

obj.d = 4;

alert(JSON.stringify(obj));
alert(JSON.stringify(objCopy));

Summary

  • ... can represent rest parameters or spread syntax.
  • Rest parameters gather arguments into an array.
  • Spread syntax expands arrays or iterables into arguments.
  • Rest parameters must appear at the end of parameters.
  • The old arguments object is array-like but not a real array.
  • Spread syntax is often used to merge arrays or copy arrays/objects.
