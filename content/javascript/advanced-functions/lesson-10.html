Arrow Functions Revisited

Arrow functions are not just a shorter syntax for writing functions. They have several special behaviors that make them very useful in many JavaScript situations.

They are especially helpful when we want to create small functions that run in another place but keep the current context.

Where Arrow Functions Are Commonly Used

JavaScript often requires small functions that are passed somewhere else to be executed later.

  • arr.forEach(func) – executes the function for every array item.
  • setTimeout(func) – executes the function later using the scheduler.
  • Many JavaScript APIs use callback functions.

Arrow Functions Have No "this"

Arrow functions do not have their own this.

Instead, they use the this value from the surrounding (outer) scope. 


let group = {

  title: "Our Group",

  students: ["John", "Pete", "Alice"],

  showList() {

    this.students.forEach(

      student => alert(this.title + ": " + student)

    );

  }

};

group.showList();

Regular Function Problem

If we used a regular function instead of an arrow function, the context would be lost. 


let group = {

  title: "Our Group",

  students: ["John", "Pete", "Alice"],

  showList() {

    this.students.forEach(function(student) {

      alert(this.title + ": " + student);

    });

  }

};

The error occurs because forEach calls the function with this = undefined.

Arrow Functions Cannot Be Constructors

Arrow functions do not have their own this, so they cannot be used with new. 


let func = () => {};

new func(); // Error

Arrow Functions vs bind

There is an important difference between arrow functions and using .bind(this).

  • .bind(this) creates a new function with fixed context.
  • Arrow functions do not bind anything; they simply inherit this from the outer scope.

Arrow Functions Have No arguments

Arrow functions also do not have their own arguments object.

They inherit it from the outer function if needed.

Example: Delaying Function Calls

Arrow functions are very useful when forwarding calls and preserving the current context. 


function defer(f, ms) {

  return function() {

    setTimeout(() => f.apply(this, arguments), ms);

  };

}

function sayHi(who) {

  alert("Hello, " + who);

}

let sayHiDeferred = defer(sayHi, 2000);

sayHiDeferred("John");

Without Arrow Functions

Without arrow functions, we would need extra variables to store this and arguments. 


function defer(f, ms) {

  return function(...args) {

    let ctx = this;

    setTimeout(function() {

      return f.apply(ctx, args);

    }, ms);

  };

}

Summary

  • Arrow functions do not have their own this.
  • They inherit this from the outer scope.
  • Arrow functions do not have an arguments object.
  • They cannot be used as constructors with new.
  • They are best used for small functions that should use the surrounding context.
