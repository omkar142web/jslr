Variable Scope and Closure

JavaScript is a very function-oriented language. Functions can be created anywhere, passed as arguments, and executed later in completely different parts of the code.

Functions can also access variables outside their own scope. In this chapter we will learn how this works using scope, lexical environments, and closures.

Variable Declarations

JavaScript provides three ways to declare variables:

  • let
  • const
  • var (older method)

In modern JavaScript we usually use let and const. The behavior discussed here applies to both.

Block Scope

Variables declared inside a code block { } are only visible inside that block. 


{
  let message = "Hello";
  alert(message);
}

alert(message); // Error

Blocks allow us to isolate variables so they do not conflict with variables outside the block.

Example with Multiple Blocks


{
  let message = "Hello";
  alert(message);
}

{
  let message = "Goodbye";
  alert(message);
}

Scope in if Statements


if (true) {
  let phrase = "Hello!";
  alert(phrase);
}

alert(phrase); // Error

Scope in Loops


for (let i = 0; i < 3; i++) {
  alert(i);
}

alert(i); // Error

Nested Functions

A function defined inside another function is called a nested function.

Nested functions can access variables from their outer function. 


function sayHiBye(firstName, lastName) {

  function getFullName() {
    return firstName + " " + lastName;
  }

  alert("Hello, " + getFullName());
  alert("Bye, " + getFullName());

}

Returning Nested Functions


function makeCounter() {
  let count = 0;

  return function() {
    return count++;
  };
}

let counter = makeCounter();

alert(counter());
alert(counter());
alert(counter());

Even after makeCounter() finishes executing, the returned function still has access to the variable count.

Lexical Environment

Every function, code block, and script has an internal object called a Lexical Environment.

This object stores variables and references to outer environments.

  • Environment Record – stores local variables.
  • Reference to the outer environment.

Function Declarations

Function declarations are fully initialized when the lexical environment is created.

This means functions declared using function can be used before their declaration.

Inner and Outer Environments

When a function runs, a new lexical environment is created for that call.

If a variable is not found locally, JavaScript searches the outer environments until it reaches the global environment.

Closures

A closure is a function that remembers its outer variables and can access them later.

In JavaScript, all functions automatically behave as closures.

  • Functions remember the environment where they were created.
  • They access variables from that environment even after execution ends.

Garbage Collection

Normally, lexical environments are removed from memory after a function finishes executing.

However, if a nested function still references that environment, it stays in memory. 


function f() {
  let value = 123;

  return function() {
    alert(value);
  }
}

let g = f();

g = null;

Engine Optimizations

JavaScript engines often optimize memory usage by removing unused variables.

Sometimes this optimization can affect debugging, especially in Chrome's V8 engine.

Summary

  • Variables declared with let and const are block-scoped.
  • Nested functions can access variables from their outer scope.
  • Each function execution creates a new lexical environment.
  • Variables are stored inside environment records.
  • A closure is a function that remembers outer variables.
  • Closures keep lexical environments alive while they are still referenced.
  • JavaScript engines may optimize unused variables internally.
