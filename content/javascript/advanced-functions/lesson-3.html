The Old "var"

This chapter explains the old var keyword. It is mainly useful for understanding older JavaScript code.

Modern JavaScript uses let and const instead.

Variable Declarations

JavaScript provides three ways to declare variables:

  • let
  • const
  • var

The var keyword is the oldest method. It behaves differently from modern declarations.

Basic Example


var message = "Hi";
alert(message);

No Block Scope

Variables declared with var do not respect block scope.

They are either function-scoped or global-scoped. 


if (true) {
  var test = true;
}

alert(test); // true
If let were used instead, the variable would only exist inside the block.

Example with let


if (true) {
  let test = true;
}

alert(test); // Error

Behavior in Loops


for (var i = 0; i < 10; i++) {
  var one = 1;
}

alert(i);   // 10
alert(one); // 1

Function Scope

When declared inside a function, var becomes scoped to the function. 


function sayHi() {

  if (true) {
    var phrase = "Hello";
  }

  alert(phrase);

}

sayHi();
alert(phrase); // Error

Redeclarations

Variables declared with var can be redeclared without error. 


var user = "Pete";

var user = "John";

alert(user); // John

Hoisting

var declarations are processed when the function begins executing.

This behavior is called hoisting. 


function sayHi() {

  phrase = "Hello";

  alert(phrase);

  var phrase;

}

sayHi();

How Hoisting Works


function sayHi() {

  var phrase;

  phrase = "Hello";

  alert(phrase);

}

sayHi();

Assignment is Not Hoisted


function sayHi() {

  alert(phrase);

  var phrase = "Hello";

}

sayHi();

The declaration is hoisted, but the assignment happens where it appears in the code.

Immediately Invoked Function Expressions (IIFE)

Before block scope existed, developers used a pattern called IIFE to create private variables. 


(function() {

  var message = "Hello";

  alert(message);

})();

Other IIFE Variations


(function() {
  alert("Parentheses around function");
})();

(function() {
  alert("Parentheses around whole expression");
}());

!function() {
  alert("Bitwise NOT operator");
}();

+function() {
  alert("Unary plus operator");
}();

Summary

  • var is an older way to declare variables.
  • It does not respect block scope.
  • It is scoped to the current function or global scope.
  • Variables declared with var can be redeclared.
  • var declarations are hoisted to the top of the function.
  • IIFE was historically used to emulate block scope.
  • Modern JavaScript prefers let and const.
