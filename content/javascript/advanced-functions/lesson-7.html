Scheduling: setTimeout and setInterval

Sometimes we want a function to run later instead of immediately. This is called scheduling a call.

JavaScript environments such as browsers and Node.js provide two methods for scheduling:

  • setTimeout – run a function once after a delay.
  • setInterval – run a function repeatedly at fixed intervals.

setTimeout

The setTimeout function schedules a function to run once after a specified delay. 


let timerId = setTimeout(func|code, delay, arg1, arg2, ...);
  • func|code – function (or code string) to execute.
  • delay – delay in milliseconds (1000ms = 1 second).
  • arg1, arg2... – arguments passed to the function.

Basic Example


function sayHi() {
  alert("Hello");
}

setTimeout(sayHi, 1000);

Passing Arguments


function sayHi(phrase, who) {
  alert(phrase + ", " + who);
}

setTimeout(sayHi, 1000, "Hello", "John");

Using Arrow Functions

Instead of passing code as a string (which is not recommended), use arrow functions. 


setTimeout(() => alert("Hello"), 1000);

Common Mistake

Do not call the function when passing it to setTimeout. 


// ❌ wrong
setTimeout(sayHi(), 1000);

// ✔ correct
setTimeout(sayHi, 1000);

Canceling a Timeout

The function returns a timer identifier that can be used to cancel the scheduled execution. 


let timerId = setTimeout(() => alert("never happens"), 1000);

clearTimeout(timerId);

setInterval

The setInterval function repeatedly executes a function after a fixed delay. 


let timerId = setInterval(func, delay);

setInterval Example


let timerId = setInterval(() => alert("tick"), 2000);

setTimeout(() => {
  clearInterval(timerId);
  alert("stop");
}, 5000);

Nested setTimeout

Another way to repeat actions is using nested setTimeout.

It allows better control over scheduling compared to setInterval. 


let timerId = setTimeout(function tick() {

  alert("tick");

  timerId = setTimeout(tick, 2000);

}, 2000);

Example: Dynamic Delay

Nested timeouts allow dynamic delays depending on the result of previous executions. 


let delay = 5000;

let timerId = setTimeout(function request() {

  // send request

  if (serverOverloaded) {
    delay *= 2;
  }

  timerId = setTimeout(request, delay);

}, delay);

Zero Delay Timeout

Calling setTimeout(func, 0) schedules a function to run after the current script finishes. 


setTimeout(() => alert("World"));

alert("Hello");

Important Notes

  • Timers may not run exactly on time.
  • Heavy CPU usage can delay timers.
  • Inactive browser tabs may slow timers.
  • Battery saving modes can increase timer delays.

Garbage Collection

Functions passed to timers remain in memory until they run or are canceled.

This also keeps their outer variables alive in memory.

Summary

  • setTimeout runs a function once after a delay.
  • setInterval runs a function repeatedly at fixed intervals.
  • Use clearTimeout and clearInterval to cancel timers.
  • Nested setTimeout can replace setInterval for more control.
  • setTimeout(func, 0) runs a function after the current script finishes.
  • Timer delays are not always exact.
