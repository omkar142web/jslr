JavaScript WeakMap and WeakSet

WeakMap and WeakSet are special collections that store objects with weak references. If an object becomes unreachable elsewhere in the program, JavaScript can automatically remove it from these collections during garbage collection.

Garbage Collection Refresher

Reachable Values

JavaScript keeps values in memory while they are reachable. If no references to an object exist, the garbage collector removes it from memory. 


let john = { name: "John" };

john = null;

// the object will be removed from memory

Objects Stored in Collections

Objects Stay Alive in Collections

If an object is stored inside another structure (like an array or map), it remains in memory while that structure exists. 


let john = { name: "John" };

let array = [john];

john = null;

// object still exists in memory
// accessible via array[0]

WeakMap

What is WeakMap?

WeakMap is similar to Map but with one major difference: its keys must be objects and they are held with weak references. 


let weakMap = new WeakMap();

let obj = {};

weakMap.set(obj,"ok");

// Error
weakMap.set("test","whoops");
Primitive values cannot be used as keys in WeakMap.

Automatic Garbage Collection


let john = { name: "John" };

let weakMap = new WeakMap();

weakMap.set(john,"...");

john = null;

// object is automatically removed

If the object has no other references, it disappears from memory and from the WeakMap.

WeakMap Methods

Method Description
weakMap.set(key,value) Add key/value pair
weakMap.get(key) Get value by key
weakMap.has(key) Check if key exists
weakMap.delete(key) Remove entry
WeakMap does not support iteration methods such as keys(), values(), or entries().

Use Case: Additional Data Storage

Attach Data to Objects

WeakMap is useful when you want to associate additional data with an object that should disappear automatically once the object is gone. 


let visitsCountMap = new WeakMap();

function countUser(user){

let count = visitsCountMap.get(user) || 0;

visitsCountMap.set(user,count + 1);

}

Use Case: Caching

Cache Results

WeakMap can store cached results for objects without worrying about memory leaks. 


let cache = new WeakMap();

function process(obj){

if(!cache.has(obj)){

let result = /* heavy calculation */ obj;

cache.set(obj,result);

return result;

}

return cache.get(obj);

}

WeakSet

What is WeakSet?

WeakSet works like Set but only stores objects and removes them automatically once they become unreachable.

Method Description
weakSet.add(value) Add object
weakSet.has(value) Check membership
weakSet.delete(value) Remove object

WeakSet Example


let visitedSet = new WeakSet();

let john = { name:"John" };
let pete = { name:"Pete" };
let mary = { name:"Mary" };

visitedSet.add(john);
visitedSet.add(pete);

alert(visitedSet.has(john));

john = null;

// automatically removed later

WeakSet is often used to track whether objects have already been processed.

Summary

  • WeakMap stores key/value pairs where keys must be objects.
  • If an object key becomes unreachable, it is automatically removed.
  • WeakMap supports only set, get, has, and delete.
  • WeakMap cannot be iterated and has no size property.
  • WeakSet stores unique objects with weak references.
  • Objects disappear from WeakSet automatically after garbage collection.
  • WeakMap and WeakSet are mainly used as secondary storage attached to objects.
