Object Methods, "this"

Objects are usually created to represent entities of the real world, like users, orders and so on.

Objects Represent Real Entities


let user = {
  name: "John",
  age: 30
};

In the real world, a user can act: select something from the shopping cart, login, logout etc.

Actions are represented in JavaScript by functions in properties.

Method Examples

For a start, let’s teach the user to say hello: 


let user = {
  name: "John",
  age: 30
};

user.sayHi = function() {
  alert("Hello!");
};

user.sayHi(); // Hello!

Here we used a Function Expression to create a function and assign it to the property user.sayHi.

Then we can call it as user.sayHi(). The user can now speak.

A function that is a property of an object is called its method.

So here we have a method sayHi of the object user.

Using a Predeclared Function

We could also use a pre-declared function as a method. 


let user = {
  // ...
};

function sayHi() {
  alert("Hello!");
}

user.sayHi = sayHi;

user.sayHi(); // Hello!

Object-Oriented Programming

When we write our code using objects to represent entities, that’s called object-oriented programming, in short: OOP.

OOP is a big topic and an interesting science of its own.

It involves deciding how to choose the right entities and organize interaction between them.

This is called software architecture and there are many famous books about it, such as:

  • Design Patterns: Elements of Reusable Object-Oriented Software
  • Object-Oriented Analysis and Design with Applications

Method Shorthand

There exists a shorter syntax for methods in an object literal. 


user = {
  sayHi: function() {
    alert("Hello");
  }
};

user = {
  sayHi() {
    alert("Hello");
  }
};

Here we omit the word function and simply write the method name followed by parentheses.

Although there are subtle differences related to inheritance, in almost all cases the shorter syntax is preferred.

“this” in Methods

It’s common that an object method needs to access information stored inside the object.

To access the object, a method can use the this keyword.

The value of this is the object before the dot used to call the method. 


let user = {
  name: "John",
  age: 30,

  sayHi() {
    alert(this.name);
  }

};

user.sayHi(); // John

During the execution of user.sayHi(), the value of this will be the object user.

Using Object Name Instead of this


let user = {
  name: "John",
  age: 30,

  sayHi() {
    alert(user.name);
  }

};

This approach is unreliable.

If we copy the object to another variable and change the original reference, the method may fail. 


let user = {
  name: "John",
  age: 30,

  sayHi() {
    alert(user.name);
  }

};

let admin = user;
user = null;

admin.sayHi(); // Error

“this” is Not Bound

In JavaScript, the keyword this behaves differently compared to many other programming languages.

It can be used in any function, even if it is not part of an object. 


function sayHi() {
  alert(this.name);
}

The value of this is evaluated at runtime depending on how the function is called.

Same Function, Different Objects


let user = { name: "John" };
let admin = { name: "Admin" };

function sayHi() {
  alert(this.name);
}

user.f = sayHi;
admin.f = sayHi;

user.f();  // John
admin.f(); // Admin

admin['f'](); // Admin

If obj.f() is called, then inside the function this equals obj.

Calling Without an Object


function sayHi() {
  alert(this);
}

sayHi(); // undefined

In strict mode, this becomes undefined.

In non-strict mode, it refers to the global object (window in browsers).

Usually calling such a function is a programming mistake.

Arrow Functions Have No “this”

Arrow functions are special because they do not have their own this.

Instead, they inherit this from the outer function. 


let user = {
  firstName: "Ilya",
  sayHi() {
    let arrow = () => alert(this.firstName);
    arrow();
  }
};

user.sayHi(); // Ilya

Summary

  • Functions stored inside object properties are called methods.
  • Methods allow objects to perform actions.
  • Methods can access object properties using this.
  • The value of this is determined at runtime.
  • When called as object.method(), the value of this becomes the object.
  • Functions can be reused across multiple objects.
  • Arrow functions do not have their own this and instead inherit it from the outer context.
