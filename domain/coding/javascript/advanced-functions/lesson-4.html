Global Object

The global object provides variables and functions that are available everywhere in the program.

These include built-in language features and environment-specific APIs.

What is the Global Object?

The global object stores values that can be accessed anywhere in the program.

  • In browsers it is called window.
  • In Node.js it is called global.
  • The standardized universal name is globalThis.

The globalThis variable works across all environments and is recommended for universal code.

Accessing Global Properties

All properties of the global object can be accessed directly. 


alert("Hello");

// same as

window.alert("Hello");

Global Variables with var

In browsers, variables declared with var automatically become properties of the global object. 


var gVar = 5;

alert(window.gVar); // 5

Function declarations behave the same way when written in the global scope.

Variables Declared with let

Variables declared with let or const do not become properties of the global object. 


let gLet = 5;

alert(window.gLet); // undefined

Creating Global Variables Explicitly

If a value should truly be global, we can explicitly assign it to the global object. 


// create global variable

window.currentUser = {
  name: "John"
};

// access it anywhere

alert(currentUser.name);


// safer access

alert(window.currentUser.name);

Avoid Too Many Global Variables

Using many global variables is generally discouraged.

Programs are easier to understand when functions receive inputs and return results instead of relying on global variables.

Using the Global Object for Polyfills

The global object can be used to check if modern language features exist.

If they are missing, we can provide a polyfill implementation. 


if (!window.Promise) {
  alert("Your browser is really old!");
}


if (!window.Promise) {

  window.Promise = ... // custom implementation

}

Summary

  • The global object stores variables accessible everywhere.
  • Built-in objects like Array and browser APIs like window.innerHeight belong to it.
  • The universal name for the global object is globalThis.
  • Environment-specific names include window (browser) and global (Node.js).
  • Variables declared with var in global scope become properties of the global object.
  • Variables declared with let and const do not.
  • Global variables should be minimized for better code design.
