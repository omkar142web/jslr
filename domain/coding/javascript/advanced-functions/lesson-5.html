Function Object, NFE

In JavaScript, functions are values. Every value has a type, and functions are actually objects.

That means functions can be called, but also treated like objects with properties.

Functions Are Objects

Functions in JavaScript are special objects that can be executed.

They can also store properties, be passed as values, or modified like any other object.

The name Property

Every function has a name property that stores the function’s name. 


function sayHi() {
  alert("Hi");
}

alert(sayHi.name); // sayHi

Automatic Function Names

JavaScript automatically assigns names to functions based on their context. 


let sayHi = function() {
  alert("Hi");
};

alert(sayHi.name); // sayHi


function f(sayHi = function(){}) {
  alert(sayHi.name); // sayHi
}

f();

Method Names


let user = {

  sayHi() {
    // ...
  },

  sayBye: function() {
    // ...
  }

};

alert(user.sayHi.name);  // sayHi
alert(user.sayBye.name); // sayBye

Functions Without Names


let arr = [function() {}];

alert(arr[0].name); // empty string

Sometimes JavaScript cannot determine the correct name, so the name property is empty.

The length Property

Functions also have a length property that shows the number of parameters in the function definition. 


function f1(a) {}
function f2(a, b) {}
function many(a, b, ...more) {}

alert(f1.length); // 1
alert(f2.length); // 2
alert(many.length); // 2

Using length for Polymorphism


function ask(question, ...handlers) {

  let isYes = confirm(question);

  for (let handler of handlers) {

    if (handler.length == 0) {

      if (isYes) handler();

    } else {

      handler(isYes);

    }

  }

}

ask("Question?",
  () => alert("You said yes"),
  result => alert(result)
);

Custom Function Properties

Functions can have custom properties just like objects. 


function sayHi() {

  alert("Hi");

  sayHi.counter++;

}

sayHi.counter = 0;

sayHi();
sayHi();

alert(\`Called \${sayHi.counter} times\`);

Properties vs Variables

Function properties are not the same as local variables.

They exist independently from variables declared inside the function.

Using Properties Instead of Closures


function makeCounter() {

  function counter() {
    return counter.count++;
  }

  counter.count = 0;

  return counter;

}

let counter = makeCounter();

alert(counter());
alert(counter());

Named Function Expression (NFE)

A Named Function Expression is a function expression that includes a name. 


let sayHi = function func(who) {
  alert(\`Hello, \${who}\`);
};

Why Use NFE?

  • Allows the function to reference itself internally.
  • The internal name is not visible outside the function.

Recursive Call with NFE


let sayHi = function func(who) {

  if (who) {
    alert(\`Hello, \${who}\`);
  } else {
    func("Guest");
  }

};

sayHi();

Problem Without NFE


let sayHi = function(who) {

  if (who) {
    alert(\`Hello, \${who}\`);
  } else {
    sayHi("Guest");
  }

};

let welcome = sayHi;

sayHi = null;

welcome(); // Error

Fixed with NFE


let sayHi = function func(who) {

  if (who) {
    alert(\`Hello, \${who}\`);
  } else {
    func("Guest");
  }

};

let welcome = sayHi;
sayHi = null;

welcome(); // Works

Summary

  • Functions in JavaScript are objects.
  • They have built-in properties like name and length.
  • Functions can also store custom properties.
  • A Named Function Expression (NFE) is a function expression with a name.
  • The internal name allows the function to call itself reliably.
  • This technique is useful for recursion and safer function references.
