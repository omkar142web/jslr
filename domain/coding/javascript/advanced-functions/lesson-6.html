The "new Function" Syntax

JavaScript provides another way to create functions using the new Function constructor.

It is rarely used in normal development, but sometimes there is no alternative.

Syntax

The syntax for creating a function dynamically is: 


let func = new Function([arg1, arg2, ...argN], functionBody);

The function is created with parameters arg1...argN and the body defined by functionBody.

Example with Parameters


let sum = new Function('a', 'b', 'return a + b');

alert(sum(1, 2)); // 3

Example Without Parameters


let sayHi = new Function('alert("Hello")');

sayHi(); // Hello

Functions Created from Strings

The main difference from other function creation methods is that the function code is provided as a string and evaluated at runtime.

Other methods require writing the function code directly inside the script.

Dynamic Code Execution

The new Function syntax allows converting strings into executable functions.

This can be useful when code is received dynamically, for example from a server. 


let str = "...code received from server...";

let func = new Function(str);

func();

Closures and new Function

Normally, functions remember the environment in which they were created using the internal property [[Environment]].

However, functions created with new Function always reference the global environment instead.

Example: No Access to Outer Variables


function getFunc() {

  let value = "test";

  let func = new Function('alert(value)');

  return func;

}

getFunc()(); // Error: value is not defined

Regular Function Behavior


function getFunc() {

  let value = "test";

  let func = function() {
    alert(value);
  };

  return func;

}

getFunc()(); // "test"

Why It Works This Way

Before code is deployed, it is often compressed using minifiers.

Minifiers rename local variables to shorter names to reduce file size.

If functions created with new Function could access outer variables, those renamed variables might break the code.

Therefore, new Function only accesses global variables.

Passing Data to new Function

To provide data to functions created with new Function, we should use parameters instead of relying on outer variables.

Alternative Syntax Forms


new Function('a', 'b', 'return a + b');

new Function('a,b', 'return a + b');

new Function('a , b', 'return a + b');

All of these forms create the same function.

Summary

  • new Function allows creating functions dynamically from strings.
  • The syntax is new Function(arg1, arg2, ..., functionBody).
  • Functions created this way use the global environment.
  • They cannot access outer variables from where they were created.
  • This design prevents problems with minified code.
  • Parameters should be used to pass values to these functions.
