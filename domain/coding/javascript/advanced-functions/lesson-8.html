Decorators and Forwarding, call/apply

JavaScript provides powerful flexibility when working with functions.

Functions can be passed around, wrapped, modified, and extended without changing the original function. This concept is called a decorator.

Transparent Caching

Imagine we have a CPU-heavy function whose result always stays the same for the same input.

Instead of recalculating the result every time, we can cache it using a decorator. 


function slow(x) {
  alert("Called with " + x);
  return x;
}

function cachingDecorator(func) {

  let cache = new Map();

  return function(x) {

    if (cache.has(x)) {
      return cache.get(x);
    }

    let result = func(x);

    cache.set(x, result);

    return result;

  };

}

slow = cachingDecorator(slow);

alert(slow(1));
alert("Again: " + slow(1));

What is a Decorator?

A decorator is a function that takes another function and modifies its behavior.

It wraps the original function with additional functionality like caching, logging, validation, or timing.

Problem with Object Methods

Decorators may break methods that rely on this.

This happens because the wrapper calls the function without the original context. 


let worker = {

  someMethod() {
    return 1;
  },

  slow(x) {
    return x * this.someMethod();
  }

};

let func = worker.slow;

func(2); // this becomes undefined

Using func.call

The call method allows us to explicitly set the value of this. 


func.call(context, arg1, arg2, ...);


function sayHi() {
  alert(this.name);
}

let user = { name: "John" };
let admin = { name: "Admin" };

sayHi.call(user);
sayHi.call(admin);

Fixing the Decorator


function cachingDecorator(func) {

  let cache = new Map();

  return function(x) {

    if (cache.has(x)) {
      return cache.get(x);
    }

    let result = func.call(this, x);

    cache.set(x, result);

    return result;

  };

}

Handling Multiple Arguments

If a function has multiple parameters, we need to create a unique cache key for the combination of arguments. 


function cachingDecorator(func, hash) {

  let cache = new Map();

  return function() {

    let key = hash(arguments);

    if (cache.has(key)) {
      return cache.get(key);
    }

    let result = func.call(this, ...arguments);

    cache.set(key, result);

    return result;

  };

}

func.apply

The apply method works similarly to call, but accepts arguments as an array-like object. 


func.apply(context, args);


func.call(context, ...args);
func.apply(context, args);

Both perform the same task, but apply receives arguments as an array-like structure.

Call Forwarding

Passing all arguments and context to another function is called call forwarding. 


let wrapper = function() {
  return func.apply(this, arguments);
};

Method Borrowing

Sometimes we borrow a method from one object and use it on another object. 


function hash() {
  return [].join.call(arguments);
}

Here we borrow the join method from arrays and use it with the arguments object.

Decorators and Function Properties

When we decorate a function, the wrapper replaces the original function.

This means properties on the original function are not automatically preserved.

More advanced decorators can preserve properties using Proxy.

Summary

  • A decorator wraps a function to modify its behavior.
  • Decorators allow adding features without changing original code.
  • func.call(context, ...args) calls a function with a specified this.
  • func.apply(context, args) works similarly but uses an array-like argument list.
  • Call forwarding passes arguments and context to another function.
  • Method borrowing allows using methods from one object on another.
