Methods of Primitives

JavaScript allows primitives such as strings and numbers to behave like objects when accessing properties or methods.

Primitives vs Objects

Primitive Values

A primitive is a value of a primitive type.

Primitive Types
string
number
bigint
boolean
symbol
null
undefined

Objects

Objects can store multiple values as properties and may contain functions. 


let john = {
name:"John",
sayHi:function(){
alert("Hi buddy!");
}
};

john.sayHi();

Objects Are Heavier

Objects require additional resources because they store properties, methods and internal structures. Primitives are lightweight and faster.

Primitive as an Object

The JavaScript Solution

JavaScript allows primitives to access methods by temporarily wrapping them inside special objects.

  • Primitives remain lightweight values.
  • Special wrapper objects provide methods.
  • The wrapper object is created temporarily and destroyed.

Wrapper Objects

Primitive Wrapper Object
string String
number Number
boolean Boolean
symbol Symbol
bigint BigInt

Example: String Method


let str = "Hello";

alert(str.toUpperCase());

The primitive string temporarily becomes a String object so that the method can run.

What Happens Internally

When calling a method on a primitive:

  • A temporary wrapper object is created.
  • The method runs on that object.
  • The object is destroyed.

Number Methods


let n = 1.23456;

alert(n.toFixed(2));

The method rounds the number to the given precision.

Constructors for Wrappers

JavaScript also has constructors like String, Number, and Boolean. They can create wrapper objects, but using them with new is discouraged. 


alert(typeof 0);

alert(typeof new Number(0));

Objects Are Always Truthy


let zero = new Number(0);

if(zero){
alert("zero is truthy!?!");
}
Objects are always truthy in JavaScript, even if they contain values like 0 or false.

Type Conversion Using Constructors

Using String(), Number(), and Boolean() without new is safe and commonly used for type conversion. 


let num = Number("123");

null and undefined

The primitives null and undefined do not have wrapper objects and do not support methods. 


alert(null.test);
Accessing properties on null or undefined results in an error.

Summary

  • Primitives are lightweight values.
  • JavaScript temporarily wraps primitives in objects to access methods.
  • Wrapper objects include String, Number, Boolean, Symbol and BigInt.
  • Using constructors with new creates objects and is discouraged.
  • Using constructors without new converts values between types.
  • null and undefined do not provide methods.
