JavaScript Iterables

Iterable objects are a generalization of arrays. They allow any object to be used in a for..of loop if it implements the proper iteration protocol.

What are Iterables?

General Idea

Iterable objects are objects that can be looped over with for..of. Arrays are iterable by default, but many other built-in objects such as strings are iterable as well.

Why Iterables?

Sometimes an object represents a collection of values (for example a list or set), but it is not technically an array. Making it iterable allows us to loop through its values easily using for..of.

Symbol.iterator

Iteration Protocol

To make an object iterable, it must implement the special method Symbol.iterator. When a for..of loop starts, JavaScript calls this method to obtain an iterator object.

Range Object Example


let range = {
  from: 1,
  to: 5
};

range[Symbol.iterator] = function() {
  return {
    current: this.from,
    last: this.to,

    next() {
      if (this.current <= this.last) {
        return { done: false, value: this.current++ };
      } else {
        return { done: true };
      }
    }
  };
};

for (let num of range) {
  alert(num);
}

The next() method returns objects like {done:false, value:...}.

Iterator vs Iterable

Separation of Concerns

The iterable object itself does not generate values directly. Instead, calling obj[Symbol.iterator]() returns an iterator object that produces values through its next() method.

Combined Iterator and Iterable


let range = {
  from: 1,
  to: 5,

  [Symbol.iterator]() {
    this.current = this.from;
    return this;
  },

  next() {
    if (this.current <= this.to) {
      return { done:false, value:this.current++ };
    } else {
      return { done:true };
    }
  }
};

for (let num of range) {
  alert(num);
}

Here the object itself acts as the iterator.

Infinite Iterators

Endless Sequences

Iterators can generate infinite sequences. For example, a range may extend to Infinity, or an iterator may produce endless random numbers.

Such loops would run forever unless manually stopped using break.

Strings Are Iterable


for (let char of "test") {
  alert(char);
}

Surrogate Pairs


let str = "𝒳😂";

for (let char of str) {
  alert(char);
}

Each character is correctly iterated even for complex Unicode symbols.

Calling Iterator Manually


let str = "Hello";

let iterator = str[Symbol.iterator]();

while(true) {
  let result = iterator.next();

  if(result.done) break;

  alert(result.value);
}

This is rarely needed but provides more control than for..of.

Iterables vs Array-Like

Iterables

Objects implementing Symbol.iterator.

Array-Like

Objects with numeric indexes and a length property that resemble arrays. 


let arrayLike = {
  0: "Hello",
  1: "World",
  length: 2
};

This object is array-like but not iterable.

Array.from

Convert to Real Array

The method Array.from() converts iterable or array-like objects into real arrays so that array methods like push, pop, etc. can be used. 


let arrayLike = {
  0: "Hello",
  1: "World",
  length: 2
};

let arr = Array.from(arrayLike);

alert(arr.pop());

Convert Iterable


let arr = Array.from(range);

alert(arr);

Mapping Function


let arr = Array.from(range, num => num * num);

alert(arr);

Array.from with Strings


let str = "𝒳😂";

let chars = Array.from(str);

alert(chars[0]);
alert(chars[1]);
alert(chars.length);

Manual Equivalent


let chars = [];

for (let char of str) {
  chars.push(char);
}

Summary

  • Objects that can be used in for..of are called iterable.
  • Iterable objects implement the Symbol.iterator method.
  • The iterator returned by it must implement next().
  • next() returns objects of form {done,value}.
  • Strings and arrays are built-in iterable objects.
  • Objects with numeric indexes and length are called array-like.
  • Array.from() converts iterables or array-likes into real arrays.
