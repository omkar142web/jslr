Constructor, Operator "new"

The regular {...} syntax allows us to create one object. But often we need to create many similar objects, like multiple users or menu items and so on.

That can be done using constructor functions and the new operator.

Constructor Function

Constructor functions technically are regular functions. There are two conventions though:

  • They are named with capital letter first.
  • They should be executed only with the new operator.

For instance: 


function User(name) {
  this.name = name;
  this.isAdmin = false;
}

let user = new User("Jack");

alert(user.name); 
alert(user.isAdmin);

When a function is executed with new, it performs the following steps:

  • A new empty object is created and assigned to this.
  • The function body executes and usually modifies this.
  • The value of this is returned.

In other words, new User(...) does something like this internally: 


function User(name) {

  // this = {};

  this.name = name;
  this.isAdmin = false;

  // return this;
}

So let user = new User("Jack") gives the same result as: 


let user = {
  name: "Jack",
  isAdmin: false
};

Now if we want to create other users, we can call new User("Ann"), new User("Alice") and so on.

That’s the main purpose of constructors – to implement reusable object creation code.

Technically, any function (except arrow functions, because they don’t have this) can be used as a constructor.

new function() { … }

If we have many lines of code related to creation of a single complex object, we can wrap them in an immediately called constructor function. 


let user = new function() {
  this.name = "John";
  this.isAdmin = false;

  // other code for creation
};

This constructor cannot be called again because it is not saved anywhere.

This trick encapsulates code that constructs a single object without future reuse.

Constructor Mode Test: new.target

Advanced topic. This syntax is rarely used but useful for understanding how constructors work internally.

Inside a function we can check whether it was called with new or without it using new.target.

It is undefined for regular calls and equals the function if called with new. 


function User() {
  alert(new.target);
}

User();      // undefined
new User();  // function User

We can also ensure the function works correctly whether called with or without new. 


function User(name) {

  if (!new.target) {
    return new User(name);
  }

  this.name = name;
}

let john = User("John");
alert(john.name);

Return from Constructors

Usually constructors do not use a return statement.

But if a return statement exists, the rules are:

  • If return returns an object, that object becomes the result.
  • If return returns a primitive, it is ignored.

function BigUser() {

  this.name = "John";

  return { name: "Godzilla" };
}

alert(new BigUser().name);


function SmallUser() {

  this.name = "John";

  return;
}

alert(new SmallUser().name);

Omitting Parentheses

Parentheses after new may be omitted. 


let user = new User;

// same as
let user = new User();

Although this syntax is allowed, omitting parentheses is generally not considered good style.

Methods in Constructor

Constructor functions can also define methods. 


function User(name) {

  this.name = name;

  this.sayHi = function() {
    alert("My name is: " + this.name);
  };

}

let john = new User("John");

john.sayHi();

The created object looks like this: 


john = {
  name: "John",
  sayHi: function() { ... }
};

To create complex objects there is a more advanced syntax called classes, which we will study later.

Summary

  • Constructor functions are regular functions but named with a capital letter by convention.
  • They should be called using the new operator.
  • Calling with new creates a new object, assigns it to this, and returns it.
  • Constructor functions allow creation of multiple similar objects.
  • JavaScript also provides built-in constructors such as Date and Set.
