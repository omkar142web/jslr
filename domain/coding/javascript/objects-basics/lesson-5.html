Optional Chaining '?.'

The optional chaining ?. is a safe way to access nested object properties, even if an intermediate property doesn’t exist.

This is a recent addition to the language. Old browsers may need polyfills.

The “Non-Existing Property” Problem

If you’ve just started to learn JavaScript, maybe this problem hasn’t appeared yet, but it’s very common.

Imagine we have user objects that hold information about our users.

Most users have addresses in user.address with the street stored in user.address.street, but some users did not provide an address. 


let user = {}; // a user without "address" property

alert(user.address.street); // Error!

JavaScript throws an error because user.address is undefined, and accessing .street on undefined fails.

In many real cases we would prefer to receive undefined instead of an error.

Another Example from Web Development

In web development we often get elements from the DOM using methods like document.querySelector().

If the element does not exist, the method returns null. 


let html = document.querySelector('.elem').innerHTML;

If the element does not exist, accessing .innerHTML of null throws an error.

In some cases we would prefer the result to simply be null or undefined.

Traditional Solution Using Conditional Checks


let user = {};

alert(user.address ? user.address.street : undefined);

This works but it’s not elegant because user.address appears twice. 


let html = document.querySelector('.elem')
  ? document.querySelector('.elem').innerHTML
  : null;

The element search is performed twice, which is inefficient.

With deeper nesting, the code becomes even harder to read. 


let user = {};

alert(
  user.address
    ? user.address.street
      ? user.address.street.name
      : null
    : null
);

Using the && Operator


let user = {};

alert(user.address &&
      user.address.street &&
      user.address.street.name);

This avoids errors but still duplicates property names.

For example, user.address appears multiple times.

That’s why the optional chaining operator ?. was introduced.

Optional Chaining

The optional chaining ?. stops evaluation if the value before it is null or undefined.

It then returns undefined instead of throwing an error. 


let user = {};

alert(user?.address?.street);


let html = document.querySelector('.elem')?.innerHTML;


let user = null;

alert(user?.address);
alert(user?.address.street);

Do Not Overuse Optional Chaining

Use ?. only where the absence of a value is acceptable.

For example, if the user object must exist but the address may not: 


user.address?.street

Overusing optional chaining may hide programming errors.

Declared Variables Requirement

The variable before ?. must exist. 


// ReferenceError
user?.address;

Short-Circuiting


let user = null;
let x = 0;

user?.sayHi(x++);

alert(x); // 0

If the left side is null or undefined, evaluation stops immediately.

Other Variants: ?.() and ?.[]


let userAdmin = {
  admin() {
    alert("I am admin");
  }
};

let userGuest = {};

userAdmin.admin?.();
userGuest.admin?.();


let key = "firstName";

let user1 = {
  firstName: "John"
};

let user2 = null;

alert(user1?.[key]);
alert(user2?.[key]);


delete user?.name;

Cannot Be Used for Assignment


let user = null;

user?.name = "John"; // Error

Optional chaining works only for reading or deleting properties, not writing them.

Summary

  • obj?.prop – returns property if object exists.
  • obj?.[prop] – safe access using bracket notation.
  • obj.method?.() – safely calls a method if it exists.
  • Optional chaining stops evaluation if the left side is null or undefined.
  • It allows safe access to nested properties.
  • Use it carefully so it does not hide programming errors.
