Object to Primitive Conversion

What happens when objects are added obj1 + obj2, subtracted obj1 - obj2 or printed using alert(obj)?

In such situations JavaScript automatically converts objects to primitive values before performing the operation.

Automatic Conversion

JavaScript does not allow developers to customize how operators work on objects.

Unlike languages like Ruby or C++, JavaScript does not allow implementing special methods to handle operators like addition.

When operations are applied to objects, JavaScript automatically converts them to primitive values first.

After conversion, the operation is performed using the resulting primitive values.

The result of operations like obj1 + obj2 will always be a primitive value, not another object.

Why This Matters

Because of this limitation, mathematical operations with objects are rarely used in real projects.

If they appear in code, it is usually because of a programming mistake.

However, there are rare exceptions where such conversions are useful. For example subtracting two Date objects returns the time difference.

Conversion Rules

Objects can only be converted to numeric or string values.

There is no conversion to boolean because all objects are considered true in boolean contexts.

Numeric conversion happens during mathematical operations such as subtraction or multiplication.

String conversion happens when objects are displayed or concatenated with strings.

Hints

JavaScript uses conversion hints to decide which type of primitive value is expected.

There are three possible hints defined in the specification.

  • "string" – used when a string is expected.
  • "number" – used when a numeric value is expected.
  • "default" – used when JavaScript is unsure which type to use.

String Hint


alert(obj);

anotherObj[obj] = 123;

Number Hint


let num = Number(obj);

let n = +obj;

let delta = date1 - date2;

let greater = user1 > user2;

Default Hint


let total = obj1 + obj2;

if (user == 1) {
  // comparison
}

Most built-in objects treat the "default" hint the same as the "number" hint.

Conversion Algorithm

To convert an object to a primitive value JavaScript checks for specific methods in the following order:

  • Call obj[Symbol.toPrimitive](hint) if it exists.
  • If hint is "string", try obj.toString() then obj.valueOf().
  • If hint is "number" or "default", try obj.valueOf() then obj.toString().

Symbol.toPrimitive

The modern way to control conversion is by implementing a method using the Symbol.toPrimitive symbol. 


let user = {
  name: "John",
  money: 1000,

  [Symbol.toPrimitive](hint) {
    return hint == "string"
      ? "{name: \\"John\\"}"
      : this.money;
  }
};

alert(user);
alert(+user);
alert(user + 500);

toString and valueOf

Before symbols existed JavaScript used two methods for conversion: toString() and valueOf().

They still work today and provide a fallback conversion mechanism. 


let user = {
  name: "John",
  money: 1000,

  toString() {
    return "{name: \\"John\\"}";
  },

  valueOf() {
    return this.money;
  }
};

alert(user);
alert(+user);
alert(user + 500);

Simple Catch-All Conversion


let user = {
  name: "John",

  toString() {
    return this.name;
  }
};

alert(user);
alert(user + 500);

Conversion Can Return Any Primitive

Conversion methods do not have to return the exact hinted type.

The only requirement is that the returned value must be a primitive.

Further Conversions


let obj = {
  toString() {
    return "2";
  }
};

alert(obj * 2);


let obj = {
  toString() {
    return "2";
  }
};

alert(obj + 2);

Summary

  • Objects are automatically converted to primitives when needed.
  • There are three conversion hints: "string", "number", and "default".
  • JavaScript first checks Symbol.toPrimitive.
  • If not present it tries valueOf() and toString().
  • All conversion methods must return primitive values.
  • Often implementing only toString() is enough for debugging and logging purposes.
